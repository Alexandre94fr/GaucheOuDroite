14. COMPÉTENCES CDA "INVISIBLE" MAIS IMPORTANTES

Dois pouvoir expliquer

Pourquoi :
- SQLite
- Unity
- API REST (REpresentational State Transfer, voir https://www.redhat.com/fr/topics/api/what-is-a-rest-api)
- Architecture multicouche
- JWT
- Entity Framework

Ne fais PAS faire :
- microservices,
- cloud,
- Kubernetes,
- architecture complexe,
- système temps réel,
- leaderboard online,
- websocket,
- multijoueur.

Objectif réel

PAS : faire un excellent jeu
MAIS : prouver que j'ai les compétences du référentiel CDA avec un projet suffisamment propre.

Ce que le jury va probablement aimer à propos du projet

- problème réel
- motivation personnelle
- application concrète
- gameplay simple
- scope raisonnable
- sujet facile à comprendre

Il faut donc le mettre en avant dans la présentation écrite et orale.
